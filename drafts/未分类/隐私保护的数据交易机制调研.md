
## 零知识累加器

[Esha Ghosh, Olga Ohrimenko, Dimitrios Papadopoulos, Roberto Tamassia, Nikos Triandopoulos  **Zero-knowledge accumulators and set algebra** (**ASIACRYPT 2016**)](https://link.springer.com/chapter/10.1007/978-3-662-53890-6_3)
针对集合成员资格证明（set membership proof），提出零知识累加器 (zero-knowledge accumulators) 来提供额外的隐藏保证: 累加值和证明不会泄露任何关于通过元素插入/删除而动态演化的集合的信息。

[Alex Ozdemir, Riad Wahby, Barry Whitehat, Dan Boneh  **Scaling verifiable computation using efficient set accumulators** (**USENIX Security 2020**)](https://www.usenix.org/conference/usenixsecurity20/presentation/ozdemir)

Merkle tree 证明数据完整性的替代方案，通过 rsa 累加器，实现更加高效的更新证明。

[Foteini Baldimtsi, Ioanna Karantaidou, Srinivasan Raghuraman  **Oblivious accumulators** (**PKC 2024**)](https://link.springer.com/chapter/10.1007/978-3-031-57722-2_4)

实现了累加器元素隐藏和添加-删除的不可区分性。

[Matteo Campanelli, Mathias Hall-Andersen, Simon Holmgaard Kamp  **Curve trees: Practical and transparent Zero-Knowledge accumulators** (**USENIX Security 2023**)](https://www.usenix.org/conference/usenixsecurity23/presentation/campanelli)

无需可信设置的累加器

[Shravan Srinivasan, Ioanna Karantaidou, Foteini Baldimtsi, Charalampos Papamanthou  **Batching, aggregation, and zero-knowledge proofs in bilinear accumulators** (**CCS 2022**)](https://dl.acm.org/doi/abs/10.1145/3548606.3560676)

基于双线性配对的高效、可聚合可批量证明的累加器。

[Jiajun Xin, Arman Haghighi, Xiangan Tian, Dimitrios Papadopoulos  **Notus: Dynamic Proofs of Liabilities from Zero-knowledge RSA Accumulators** (**USENIX Security 2024**)](https://www.usenix.org/conference/usenixsecurity24/presentation/xin)


Snark friendly 的 rsa 累加器，用于实现区块链的负债证明

[Nirvan Tyagi, Ben Fisch, Andrew Zitek, Joseph Bonneau, Stefano Tessaro  **VeRSA: Verifiable registries with efficient client audits from RSA authenticated dictionaries** (**CCS 2022**)](https://dl.acm.org/doi/abs/10.1145/3548606.3560605)


## 向量承诺

 [Weijie Wang, Annie Ulichney, Charalampos Papamanthou  **BalanceProofs: Maintainable vector commitments with fast aggregation** (**USENIX Security 2023**)](https://www.usenix.org/conference/usenixsecurity23/presentation/wang-weijie)

[Srinivasan, Shravan, Chepurnoy, Alexander, Charalampos Papamanthou, Tomescu, Alin, Yupeng Zhang  **Hyperproofs: Aggregating and Maintaining Proofs in Vector Commitments** (**USENIX Security 2022**)](https://www.usenix.org/conference/usenixsecurity22/presentation/srinivasan)

 [Christodoulos Pappas, Dimitrios Papadopoulos, Charalampos Papamanthou  **HydraProofs: Optimally Computing All Proofs in a Vector Commitment with applications to efficient zkSNARKs over data from multiple users** (**S&P 2025**)](https://www.computer.org/csdl/proceedings-article/sp/2025/223600d180/26hiVmh7o9q)

线性时间生成全体开证明（OpenAll）”的向量承诺（VC），即对长度为 N 的向量，一次性生成所有位置的打开证明总时间为 O (N)。同时，该方案可直接与一类将数据编码为多线性多项式的 zkSNARK（如 GKR 系列）对接，允许在不逐项打开承诺的情况下，对承诺的原像直接运行证明器
