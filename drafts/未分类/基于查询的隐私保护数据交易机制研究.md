[Taking authenticated range queries to arbitrary dimensions (**CCS 2014**)](https://dl.acm.org/doi/abs/10.1145/2660267.2660373)

外包数据库上认证多维范围查询 (authenticated multi-dimensional range queries) 问题，实现了：(i) 与维度数量呈线性扩展关系，(ii) 支持任意维度集合上的查询，且设置成本和存储开销与属性数量呈线性关系。

[Zero-knowledge accumulators and set algebra (**ASIACRYPT 2016**)](https://link.springer.com/chapter/10.1007/978-3-662-53890-6_3)

针对集合成员资格证明（set membership proof），提出零知识累加器 (zero-knowledge accumulators) 来提供额外的隐藏保证: 累加值和证明不会泄露任何关于通过元素插入/删除而动态演化的集合的信息。

[An expressive zero-knowledge set accumulator (**EuroS&P 2017**)](https://ieeexplore.ieee.org/document/7961978/)]

提出 expressive 集合累加器，为大量累积集合上的运算提供了简明的证明，包括交集、并集、集合差、SUM、COUNT、MIN、MAX 和 RANGE，以及上述的任意嵌套，并扩展成零知识。

有关零知识累加器的进一步发展：

[Succinct zero-knowledge batch proofs for set accumulators (**CCS 2022**)](https://dl.acm.org/doi/abs/10.1145/3548606.3560677)

零知识：批量证明成员资格，批量更新累加器

[Batching-efficient ram using updatable lookup arguments (**CCS 2024**)](https://dl.acm.org/doi/abs/10.1145/3658644.3670356)

零知识可验证计算（如 zksnark、zkstark，zkvm 等协议）中对随机存取存储器（RAM，Random Access Memory）行为的密码学模拟，是比累加器更加高级且抽象的概念，重点在于证明某次内存读写操作在正确状态下发生。

[Notus: Dynamic Proofs of Liabilities from Zero-knowledge RSA Accumulators (**USENIX Security 2024**)](https://www.usenix.org/conference/usenixsecurity24/presentation/xin)

基于累加器设计的负债证明承诺方案，可高效更新。


[HydraProofs: Optimally Computing All Proofs in a Vector Commitment with applications to efficient zkSNARKs over data from multiple users (**S&P 2025**)](pappas2025hydraproofs)

线性时间生成全体开证明（OpenAll）”的向量承诺（VC），即对长度为 N 的向量，一次性生成所有位置的打开证明总时间为 O (N)。同时，该方案可直接与一类将数据编码为多线性多项式的 zkSNARK（如 GKR 系列）对接，允许在不逐项打开承诺的情况下，对承诺的原像直接运行证明器
